= APIkit プロジェクトの変更方法
ifndef::env-site,env-github[]
include::_attributes.adoc[]
endif::[]

次の条件下では、RAML リソースとアクションのマッピングをカスタマイズする必要があります。

* バックエンドファーストの設計プロセスの一部としてバックエンドフローを生成している。
* バックエンドフロー名を変更している。

RAML リソースとアクションのマッピングのカスタマイズは、バックエンドファーストの設計プロセスで行われます。


リソースの追加などで APIkit プロジェクトの RAML コードを変更した場合、次のようにフローを新しいリソースに関連付けるためにフローを再生成する必要があります。

プロジェクトエクスプローラで、プロジェクト名を右クリックし、[Mule] > [Generate Flows From REST API (REST API からフローを生成)] を選択します。
